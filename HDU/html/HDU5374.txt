<tr><td align=center><h1 style='color:#1A5CC8'>Tetris</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 65536/65536 K (Java/Others)<br>Total Submission(s): 443&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 236<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>Tetris<br>God Wu enjoy coding some trivial games like Tetris.His Tetris is made up by 9 columns and 12 rows.  <br><center><img style='max-width:100%;' src=../../data/images/5374-1.jpg></center><br>The game will prossess as following steps<br>1.a random token will appear ,and its special square will be on (4,9);<br>2.the player can send a signal which means up,left,right and down to control the tokens;(If the operation which means is illegal,the operation will be skipped)<br>3.the token will fall a unit.(If the fall is illegal,we prossess the 4th step.Otherwise,we prossess the 2nd step)<br>4.If there is a horizontal line of nine units without gaps,the line will disappear and the tokens above it will fall.If a line will disappear, we will get a score and prossess 4th step again.If there is not a horizontal line of nine units without gaps,we go back to 1st step  <br><br>For the sake of debugging, God Wu just created three kinds of Tetris piece. Each type of piece do the same while receiving &quot;left&quot;, &quot;right&quot; and &quot;down&quot; operation. specificly:  <br>Left: the entire piece moves left one square with the special square.  <br>Right: the entire piece moves right one square with the special square.  <br>Down: the entire piece moves down one square with the special square.  <br>Up: rotate 90 degree clockwise  <br>The original direction of each square will be same as the first pattern in each picture.  <br>More details see the following pictures  <br>and following is the detail about the &quot;up&quot; operatrion, notice the green square is the special square:  <br>1. O-type  <br><center><img style='max-width:100%;' src=../../data/images/5374-2.jpg></center><br>2. I-type  <br><center><img style='max-width:100%;' src=../../data/images/5374-3.jpg></center><br>3. J-type  <br><center><img style='max-width:100%;' src=../../data/images/5374-4.jpg></center><br>God Wu begins to test the game after finishing coding, and ran a lot of data. However, he find score record is omitted. So he ask you to finish it.<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>Multiple testcase. The first line is an integer T, refers the case number, followed by T case, each case containing three lines:<br><br>The first line is n, refers the number of falling pieces.<br><br>Next line is a string s(|s|&lt;=1000), which shows the operation God Wu made from the begining of the Game to the End of the last piece stop moving.<br><br>The third line has n integers, the ith integer ai represent the type of the ith piece.(0 is for O-type, 1 is for I-type and 2 is for J-type)<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each test case, output the case number and the score obtained from operation sequence. Format should be &quot;Case X: Y&quot;,X is the case number and Y is the score.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1
18

1 1 2 2 1 2 0 2 2 1 0 2 0 2 0 2 1 1</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">Case 1: 6
