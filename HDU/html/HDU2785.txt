<tr><td align=center><h1 style='color:#1A5CC8'>On-Line Banking</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 4000/2000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 407&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 101<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>A central bank is an institution that is responsible for the monetary policy of a country. Beside others, it monitors and supervises other banks. Your task is to write a computer program that will serve as a simple supervision instrument for monitoring bank accounts.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The input contains several scenarios. Each scenario begins with a line containing a single positive integer A, the number of bank accounts at the beginning of the supervision, 0 &lt; A â‰¤ 100. Then there are A lines each describing one account. Such a line contains the account number, one space, and a non-negative decimal number specifying the account balance.<br><br>Each of the following lines specifies a request for one bank operation. The line begins with a command and then, separated by a space, there are optional parameters. The list of commands follows, the third column in the table shows the number of parameters.<br><br><center><img style='max-width:100%;' src=../../../data/images/C201-1002-1.jpg></center><br><br>The number of requests in every scenario will always be between 0 and 1000 (inclusive). The last request of a scenario is followed by the word &quot;end&quot; and one empty line. Then the next scenario begins. The last scenario is followed by a line containing zero in place of A.<br><br>All account numbers consist of exactly four decimal digits followed by a slash character (&quot;/&quot;) and one digit specifying a code of the bank that operates the account. Each bank has its own unique code.<br><br>Amounts are always given as non-negative decimal numbers with exactly two digits after the decimal point. No amount in the input will be higher than 10 000. No unnecessary leading zeros are permitted, i.e., only amounts strictly less than 1.00 may start with a zero.<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each request, output exactly one line. The line must begin with the command followed by the amount parameter (if present) separated from the command by a space. Then there is a colon (&quot;:&quot;), one space and the result of the operation.<br><br>create: If there is already an account with the same number in the same bank, the result will be &quot;already exists&quot;. Otherwise, create the account (with the initial balance of zero) and output &quot;ok&quot;.<br><br>For all commands other than create, if there is no account with the number specified as a parameter, the result will always be &quot;no such account&quot;. If the account exists (or both accounts do), the following rules apply.<br><br>deposit: The result is always &quot;ok&quot; for existing accounts. Simply add the amount to the account balance.<br><br>withdraw: If the balance of the specified account is strictly lower than the requested amount, the result will be &quot;insufficient funds&quot;. Otherwise, subtract the amount and output &quot;ok&quot;.<br><br>transfer: First, if both account numbers are equal (including the bank code), the result should be &quot;same account&quot;. Similar to the previous case, if the balance of the source account is lower than the amount to be transferred, the result will be &quot;insufficient funds&quot;. Otherwise, transfer the money from the source account to the target one and output either &quot;ok&quot; if the transfer happens inside one bank or &quot;interbank&quot; if the money have to be transferred from one bank to another.<br><br>After each scenario, output the word &quot;end&quot; and one empty line. After the &quot;end&quot; of the last scenario, output one additional line containing the word &quot;goodbye&quot;. </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">3
