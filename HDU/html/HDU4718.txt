<tr><td align=center><h1 style='color:#1A5CC8'>The LCIS on the Tree</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 6000/3000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 65535/65535 K (Java/Others)<br>Total Submission(s): 1490&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 424<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>For a sequence S<sub>1</sub>, S<sub>2</sub>, ... , S<sub>N</sub>, and a pair of integers (i, j), if 1 &lt;= i &lt;= j &lt;= N and S<sub>i</sub> &lt; S<sub>i+1</sub> &lt; S<sub>i+2</sub> &lt; ... &lt; S<sub>j-1</sub> &lt; S<sub>j</sub> , then the sequence S<sub>i</sub>, S<sub>i+1</sub>, ... , S<sub>j</sub> is a <b>CIS</b> (Continuous Increasing Subsequence). The longest <b>CIS</b> of a sequence is called the <b>LCIS</b> (Longest Continuous Increasing Subsequence).<br>Now we consider a tree rooted at node 1. Nodes have values. We have Q queries, each with two nodes u and v. You have to find the shortest path from u to v. And write down each nodes' value on the path, from u to v, inclusive. Then you will get a sequence, and please show us the length of its <b>LCIS</b>.<br> </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The first line has a number T (T &lt;= 10) , indicating the number of test cases.<br>For each test case, the first line is a number N (N &lt;= 10<sup>5</sup>), the number of nodes in the tree.<br>The second line comes with N numbers v1, v2, v3 ... , v<sub>N</sub>, describing the value of node 1 to node N. (1 &lt;= v<sub>i</sub> &lt;= 10<sup>9</sup>)<br>The third line comes with N - 1 numbers p<sub>2</sub>, p<sub>3</sub>, p<sub>4</sub> ... , p<sub>N</sub>, describing the father nodes of node 2 to node N. Node 1 is the root and will have no father.<br>Then comes a number Q, it is the number of queries. (Q &lt;= 10<sup>5</sup>)<br>For next Q lines, each with two numbers u and v. As described above.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For test case X, output &quot;Case #X:&quot; at the first line.<br>Then output Q lines, each with an answer to the query.<br>There should be a blank line <b>*BETWEEN*</b> each test case.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1
5
1 2 3 4 5
1 1 3 3
3
1 5
4 5
2 5</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">Case #1:
3
2
3</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Source</div> <div class=panel_content> <a href="/search.php?field=problem&key=2013+ACM%2FICPC+Asia+Regional+Online+%A1%AA%A1%AA+Warmup2&source=1&searchmode=source"> 2013 ACM/ICPC Asia Regional Online —— Warmup2 </a> </div> <div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Recommend</div> <div class=panel_content>zhuyuanchen520</div><div class=panel_bottom>&nbsp;</div><br><center style='font-size:15px;font-family:Arial;font-weight:bold;color:#1A5CC8'><a href='statistic.php?pid=4718'>Statistic</a>&nbsp;|&nbsp;<a href='submit.php?pid=4718'>Submit</a>&nbsp;|&nbsp;<a href="./discuss/problem/list.php?problemid=4718">Discuss</a> | <a href='note/note.php?pid=4718'>Note</a><br></td></tr><tr>
