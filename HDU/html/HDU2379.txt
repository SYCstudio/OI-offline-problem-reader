<tr><td align=center><h1 style='color:#1A5CC8'>Dice Password Security</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 1000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 303&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 93<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>NCIM Group sponsored problem. <br><br>The NCIM Group does a lot of work on IT solutions in defense and security. Good security usually starts with picking a strong password. Generating a password at random is generally a good practice. For example, a password like &quot;2R4eZ9Rqup&quot; is a bit harder to guess than &quot;god&quot;, &quot;love&quot;, &quot;sex&quot; or &quot;secret&quot;. <br><br>The problem with passwords consisting of random letters and digits is that they are hard to remember. Instead of using letters and digits it is also possible to generate passwords by putting random words together. Words are easier to remember than letters and digits. <br><br>Using a dictionary of 7776 (6<sup>5</sup>) words, a 5-random-word password is about as strong as a 11-random-character password. <br><br>7776<sup>5</sup> = 28430288029929701376 ≈ 3 * 10<sup>19</sup><br><br>62<sup>11</sup> = 52036560683837093888 ≈ 5 * 10<sup>19</sup> <br><br>Some applications hide the password you are typing on the screen by printing dots or asterisks. This allows someone watching your screen to count the number of characters in your password. The NCIM Group wants you to find out whether or not this compromises the strength of your password. <br><br>You must write a program that calculates the number of possible passwords that can be generated given: <br><br>the dictionary of words, the amount of words used to generate the password and the length of the password. <br><br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>On the first line an integer t (1 &lt;= t &lt;= 100): the number of test cases. Then for each test case: <br><br>One line with three positive integers m (1 &lt;= m &lt;= 7776), n (1 &lt;= n &lt;= 5) and q (1 &lt;= q &lt;= 20): the number of words in the dictionary, the number of words to generate the password, and the number of queries, respectively. <br><br>The dictionary: m lines each containing one word w<sub>i</sub> Each word consists only of lowercase letters. The length of each word will be between 3 and 10 inclusive. No word in the dictionary will be a substring of another word in the dictionary. <br><br>q lines each containing a positive integer l<sub>j</sub> (1 &lt;= l<sub>j</sub> &lt;= 50), the length observed. <br><br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each test case: <br><br>q lines with: the number of possible passwords with length l<sub>j</sub> . This number will be smaller than 2<sup>63</sup>. <br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1
4 2 2
aap
noot
mies
piet
7
8</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">6
