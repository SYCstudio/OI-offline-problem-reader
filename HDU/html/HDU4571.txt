<tr><td align=center><h1 style='color:#1A5CC8'>Travel in time</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 8000/4000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 2952&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 662<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>　　Bob gets tired of playing games, leaves Alice, and travels to Changsha alone. Yuelu Mountain, Orange Island, Window of the World, the Provincial Museum etc...are scenic spots Bob wants to visit. However, his time is very limited, he can’t visit them all. <br>　　Assuming that there are N scenic spots in Changsha, Bob defines a satisfaction value Si to each spot. If he visits this spot, his total satisfaction value will plus Si. Bob hopes that within the limited time T, he can start at spot S, visit some spots selectively, and finally stop at spot E, so that the total satisfaction value can be as large as possible. It's obvious that visiting the spot will also cost some time, suppose that it takes C<sub>i</sub> units of time to visit spot i ( 0 &lt;= i &lt; N ).<br>　　Always remember, Bob can choose to pass by a spot without visiting it (including S and E), maybe he just want to walk shorter distance for saving time. <br>　　Bob also has a special need which is that he will only visit the spot whose satisfaction value is <strong>strictly</strong> larger than that of which he visited last time. For example, if he has visited a spot whose satisfaction value is 50, he would only visit spot whose satisfaction value is 51 or more then. The paths between the spots are bi-directional, of course.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>　　The first line is an integer W, which is the number of testing cases, and the W sets of data are following.<br>　　The first line of each test data contains five integers: N M T S E. N represents the number of spots, 1 &lt; N &lt; 100; M represents the number of paths, 0 &lt; M &lt; 1000; T represents the time limitation, 0 &lt; T &lt;= 300; S means the spot Bob starts from. E indicates the end spot. (0 &lt;= S, E &lt; N)<br>　　The second line of the test data contains N integers C<sub>i</sub> ( 0 &lt;= C<sub>i</sub> &lt;= T ), which means the cost of time if Bob visits the spot i.<br>　　The third line also has N integers, which means the satisfaction value Si that can be obtained by visiting the spot i ( 0 &lt;= S<sub>i</sub> &lt; 100 ).<br>　　The next M lines, each line contains three integers u v L, means there is a bi-directional path between spot u and v and it takes L units of time to walk from u to v or from v to u. (0 &lt;= u, v &lt; N, 0 &lt;= L &lt;= T)</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>　　Output case number in the first line (formatted as the sample output).<br>　　The second line contains an integer, which is the greatest satisfaction value.<br>If Bob can’t reach spot E in T units of time, you should output just a “0” (without quotation marks).</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1
4 4 22 0 3
1 1 1 1
5 7 9 12
0 1 10
1 3 10
0 2 10
2 3 10</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">Case #1:
