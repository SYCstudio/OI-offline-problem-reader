<tr><td align=center><h1 style='color:#1A5CC8'>Cave Crisis</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 4000/2000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 161&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 31<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>R2D2 was exploring a tunnel when a cave-in suddenly occurred. Oh no, is he trapped?<br><br><center><img style='max-width:100%;' src=../../../data/images/C239-1003-1.jpg></center><center><b>Figure1:</b> Overhead view of the cave crisis from the third example test case.</center><br><br>From an overhead view, we can see all the obstacles (debris) on a two-dimensional Cartesian plane. The tunnel is w cm wide, bounded by the lines <b>y = w/2</b> and <br><b>y = -w/2</b>. R2D2 starts at <b>(0, 0)</b>, and has a perfectly circular footprint of radius r. The exit of the tunnel lies to the right of the line <b>x = 1000</b>. Between R2D2 and the exit lie a number of polygonal obstacles.<br><br>Is it possible for R2D2 to navigate between the obstacles and make it to the exit?<br><br><center><img style='max-width:100%;' src=../../../data/images/C239-1003-2.jpg></center><br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The input file will contain multiple test cases. Each test case begins with a single line containing an even integer <b>w</b> (2 &lt;= <b>w</b> &lt;= 1000), the width of the tunnel, and an integer <b>N</b> (0 &lt;= <b>N</b> &lt;= 100), the number of obstacles. The next <b>N</b> lines each contain the description of a single obstacle. The <b>i</b><sup>th</sup> obstacle is a simple polygon, specified on a single line containing an integer <b>n</b><sub>i</sub> (3 &lt;= <b>n</b><sub>i</sub> &lt;= 10), the number of vertices, followed by <b>n</b><sub>i</sub> pairs of integers, <b>x</b><sub>ij</sub> and <b>y</b><sub>ij</sub> (0 &lt;= <b>x</b><sub>ij</sub> &lt;= 1000 and <b>-w</b>/2 &lt;= <b>y</b><sub>ij</sub> &lt;= <b>w</b>/2 for <b>j</b> = 1, ..., <b>n</b><sub>i</sub> ), specifying the coordinates of the vertices in counterclockwise order. Note that obstacles in the input may touch or even overlap. However, you are guaranteed that R2D2â€™s initial location will not touch or overlap any obstacle. The vertices of each polygon are unique, no two nonconsecutive edges of the polygon intersect (even at their endpoints), and each polygon is guaranteed to have nonzero area. The end-of-input is denoted by an invalid test case with <b>w</b> = <b>N</b> = 0 and should not be processed.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each input test case, you must determine the maximum radius <b>r</b> &gt; 0 that R2D2 could have and still be able to plan a path from his starting location (0, 0) to the tunnel exit without overlapping with any of the obstacles. You should print either this maximum radius <b>r</b> (rounded to two decimal places)or the message &quot;<b>impossible</b>&quot; if no such radius exists.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">6 2

0 0</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1.00
