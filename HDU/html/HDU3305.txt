<tr><td align=center><h1 style='color:#1A5CC8'>John's  Canonical Difference Bound Matrices</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 4000/2000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 115&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 0<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content> <br><br>When John studied the timed automaton, he met the problem about how to trigger the machine. With the problem deeply studied, he found that it can be ascribed to the clock constraints of the timed automaton. The timed automation in question is described below:<br><br>The clock variables, or simply clocks, are variables whose values are integers. Of course, time passes at the same rate for all clocks, and any clock can be reset to zero. John uses C to denote the finite set of clocks, and defines the clock constraints for C as follows:<br><br><br>(1)<br>    All inequalities of the form t#c or c#t are clock constraints, where t is a clock, # is either &lt; or &lt;= , and c is an integer. <br>(2)<br>    If A1 and A2 are clock constraints, then A1 ^ A2 is a clock constraint. <br><br><br>John notes that a clock constraint can define several regions in some multidimensional space. He wants to know such regions, so he defines the clock zones recursively as follows.<br><br><center><img style='max-width:100%;' src=../../../data/images/con229-1010-1.JPG></center><br><br>For simplicity, he let C0 = C <img style='max-width:100%;' src=../../../data/images/con229-1010-3.JPG> {x0} , where x0 is a reference clock whose value is always 0. The clock zone A can be described by a Difference Bound Matrix D (called a DBM) which is a matrix (Dij) of size | C0|Ã—| C0| . Each Dij has the form (dij,#) , where dij <img style='max-width:100%;' src=../../../data/images/con229-1010-2.JPG> Z<img style='max-width:100%;' src=../../../data/images/con229-1010-3.JPG> {$} , # <img style='max-width:100%;' src=../../../data/images/con229-1010-2.JPG> { &lt; ,&lt;=} . The value of Dij can be evaluated in the following form:<br><br>For every inequality xi - xj#dij in clock zone A , let Dij = (dij,#) , where xi and xj are two clocks. If the bound of xi - xj for xi and xj is unknown, let Dij = ($, &lt; ) .<br><br>For example, DBM of the clock zone given by x1 - x2 &lt; 2 ^  0 &lt; x2&lt;=2 ^ 1&lt;=x1 is shown below: <br><br><center><img style='max-width:100%;' src=../../../data/images/con229-1010-4.JPG></center><br><br>The representation of a clock zone by a DBM is not unique. In this example, there are some implied constraints that are not reflected in the DBM. Since x1 - x2 &lt; 2 and x2&lt;=2 , it must be the case x1 &lt; 4 . Since x0 = 0 , the original D10 = ($, &lt; ) can be changed into D10 = (4, &lt; ) . Such adjusting operation is called the tighten operation.<br><br>Now John wants to do the similar adjusting operations of difference bounds for all clocks xi and xj until further application of this tighten operation does not change the matrix. John obtains the following new canonical difference bound matrix: <br><br><center><img style='max-width:100%;' src=../../../data/images/con229-1010-5.JPG></center><br><br><br>Note that some clock zone may contain contrary conditions and has not canonical difference bound matrix.<br><br>But John can not obtain a canonical difference bound matrix for a complex clock zone. He asks for your help. </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The first line of the input file is a single integer T  (1&lt;=T&lt;=20) , which is the number of test cases you must process, followed by T  test cases:<br><br>Each test case consists of several lines. Four integers i , j , d and r are given on each line, representing a constraint xi - xj &lt; d or xi - xj&lt;=d (0&lt;=i, j&lt;=m, -10000 &lt; d &lt; 10000) . If r = 0 , then this line represents an inequality in the form of xi - xj &lt; d , otherwise it represents an inequality in the form of xi - xj&lt;=d . The maximal index m of clocks indicates that the indexes of the clocks are 0, 1,..., m,(1&lt;=m&lt;=100) . Note that you have to get the value of m by yourself.<br><br>A symbol # given on a single line indicates the end of a test case. </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each test case, first output ``Case #:&quot; on a single line, where # is the case number starting from 1. Print a blank line after each test case.<br><br>For each test case, output the description of the canonical difference bound matrix. If it doesn't have a canonical difference bound matrix, print ``Canonical DBM does not exist.&quot; (without quotes); If it has a such canonical difference bound matrix, print the matrix in the format as indicated in the sample output. Every element Dij of the matrix should be written in the form (dij,#) , where # is either &lt; or &lt;=. If the bound of xi - xj for xi and xj is unknown, print ($,&lt;) at the position (i, j) . Two consecutive elements on each row should be separated by a single space. </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1
