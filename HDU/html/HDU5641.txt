<tr><td align=center><h1 style='color:#1A5CC8'>King's Phone</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 65536/65536 K (Java/Others)<br>Total Submission(s): 3477&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 812<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>In a military parade, the King sees lots of new things, including an Andriod Phone. He becomes interested in the pattern lock screen.<br><br>The pattern interface is a $3 \times 3$ square lattice, the three points in the first line are labeled as $1, 2, 3$, the three points in the second line are labeled as $4, 5, 6$, and the three points in the last line are labeled as $7, 8, 9$。The password itself is a sequence, representing the points in chronological sequence, but you should follow the following rules:<br><br>- The password contains at least four points.<br><br><br>- Once a point has been passed through. It can't be passed through again.<br>  <br>- The middle point on the path can't be skipped, unless it has been passed through($3427$ is valid, but $3724$ is invalid).<br>  <br>  His password has a length for a positive integer $k (1\le k\le 9)$, the password sequence is $s_1,s_2...s_k(0\le s_{i} &lt; INT\_MAX)$ , he wants to know whether the password is valid. Then the King throws the problem to you.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The first line contains a number&amp;nbsp;$T(0 &lt; T \le 100000)$, the number of the testcases.<br><br>For each test case, there are only one line. the first first number&amp;nbsp;$k$，represent the length of the password, then $k$ numbers, separated by a space, representing the password sequence $s_1,s_2...s_k$.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>Output exactly $T$ lines. For each test case, print `valid` if the password is valid, otherwise print `invalid`</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">3
4 1 3 6 2
4 6 2 1 3
4 8 1 6 7</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">invalid
valid
valid

hint:
For test case #1：The path $1\rightarrow 3$ skipped the middle point $2$, so it's invalid.

For test case #2：The path $1\rightarrow 3$ doesn't skipped the middle point $2$, because the point 2 has been through, so it's valid.

For test case #2：The path $8\rightarrow 1 \rightarrow 6 \rightarrow 7$ doesn't have any the middle point $2$, so it's valid.</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Source</div> <div class=panel_content> <a href="/search.php?field=problem&key=BestCoder+Round+%2375&source=1&searchmode=source"> BestCoder Round #75 </a> </div> <div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Recommend</div> <div class=panel_content>wange2014</div><div class=panel_bottom>&nbsp;</div><br><center style='font-size:15px;font-family:Arial;font-weight:bold;color:#1A5CC8'><a href='statistic.php?pid=5641'>Statistic</a>&nbsp;|&nbsp;<a href='submit.php?pid=5641'>Submit</a>&nbsp;|&nbsp;<a href="./discuss/problem/list.php?problemid=5641">Discuss</a> | <a href='note/note.php?pid=5641'>Note</a><br></td></tr><tr>
