<tr><td align=center><h1 style='color:#1A5CC8'>#define is unsafe</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 621&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 394<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>Have you used #define in C/C++ code like the code below?<br><br>#include &lt;stdio.h&gt;<br>#define MAX(a , b) ((a) &gt; (b) ? (a) : (b))<br>int main()<br>{<br>&nbsp;&nbsp;printf(&quot;%d\n&quot; , MAX(2 + 3 , 4));<br>&nbsp;&nbsp;return 0;<br>}<br><br>Run the code and get an output: 5, right?<br>You may think it is equal to this code:<br><br>#include &lt;stdio.h&gt;<br>int max(a , b) {&nbsp;&nbsp;return ((a) &gt; (b) ? (a) : (b));&nbsp;&nbsp;}<br>int main()<br>{<br>&nbsp;&nbsp;printf(&quot;%d\n&quot; , max(2 + 3 , 4));<br>&nbsp;&nbsp;return 0;<br>}<br><br>But they aren't.Though they do produce the same anwser , they work in two different ways.<br>The first code, just replace the MAX(2 + 3 , 4) with ((2 + 3) &gt; (4) ? (2 + 3) : 4), which calculates (2 + 3) twice.<br>While the second calculates (2 + 3) first, and send the value (5 , 4) to function max(a , b) , which calculates (2 + 3) only once.<br><br>What about MAX( MAX(1+2,2) , 3 ) ? <br>Remember &quot;replace&quot;.<br>First replace: MAX( (1 + 2) &gt; 2 ? (1 + 2) : 2 , 3)<br>Second replace: ( ( ( 1 + 2 ) &gt; 2 ? ( 1 + 2 ) : 2 ) &gt; 3 ? ( ( 1 + 2 ) &gt; 2 ? ( 1 + 2 ) : 2 ) : 3).<br>The code may calculate the same expression many times like ( 1 + 2 ) above.<br>So #define isn't good.In this problem,I'll give you some strings, tell me the result and how many additions(加法) are computed.<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The first line is an integer T(T&lt;=40) indicating case number.<br>The next T lines each has a string(no longer than 1000), with MAX(a,b), digits, '+' only(Yes, there're no other characters).<br>In MAX(a,b), a and b may be a string with MAX(c,d), digits, '+'.See the sample and things will be clearer.<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For each case, output two integers in a line separated by a single space.Integers in output won't exceed 1000000.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">6
MAX(1,0)
1+MAX(1,0)
MAX(2+1,3)
MAX(4,2+2)
MAX(1+1,2)+MAX(2,3)
MAX(MAX(1+2,3),MAX(4+5+6,MAX(7+8,9)))+MAX(10,MAX(MAX(11,12),13))</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">1 0
