<tr><td align=center><h1 style='color:#1A5CC8'>Game of Arrays</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 65536/65536 K (Java/Others)<br>Total Submission(s): 379&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 105<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>Tweek and Craig are good friends and always playing together. And they just invented a new game when doing their math homework.<br><br>First of all, they write three arrays $A$, $B$, and $C$, each with $N$ numbers. Then on the black board, they write those arrays as<br><br>$$A + B = C$$<br><br>If the equation is satisfied, it means for each position $i$ from $1$ to $N$, there are $A_i + B_i = C_i$ holds.<br><br>Of course, this equation is not always satisfied at the very beginning.<br><br>Fortunately, for arrays $A$, $B$ and $C$, some numbers are $changeable$, while other's are not. Those $changeable$ numbers' positions are determined before the game begins.<br><br>During the game, Tweek and Craig will take turns, trying to change a number from an array. Tweek plays first.<br><br>In each turn, the player can choose a $changeable$ number from an array, and substract it by one. However, no negative numbers should appear, so the chosen number cannot be $0$ before substraction.<br><br>Tweek's goal is to make the equation satisfied during the game, while Craig's goal is to prevent it to happen.<br><br>The game ends when the equation is satisfied (a win for Tweek) or there are no possible moves but still $A + B \neq C$ (means there is <b>at least one</b> $i \in \left[1, N\right]$, where $A_i + B_i \neq C_i$, which is a win for Craig).<br><br>Given $A$, $B$ and $C$, and the position of $changeable$ numbers for each array, your task is to determine the winner.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>First line contains an integer $T$, which indicates the number of test cases.<br><br>Every test case begins with an integers $N$, which is the length of array $A$, $B$ and $C$.<br><br>The $2^{nd}$ line and $3^{rd}$ line describe the array $A$. The $2^{nd}$ line contains $N$ intergers $A_1$, $A_2$, $\cdots$, $A_N$, indicating the elements in array $A$. The $3^{rd}$ line contains $N$ intergers $u_1$, $u_2$, $\cdots$, $u_N$, and $u_i$ is $1$ if $A_i$ is $changeable$, otherwise $u_i$ is $0$.<br><br>The $4^{th}$ line and $5^{th}$ line describe the array $B$. The $4^{th}$ line contains $N$ intergers $B_1$, $B_2$, $\cdots$, $B_N$, indicating the elements in array $B$. The $5^{th}$ line contains $N$ intergers $v_1$, $v_2$, $\cdots$, $v_N$, and $v_i$ is $1$ if $B_i$ is $changeable$,otherwise $v_i$ is $0$.<br><br>The $6^{th}$ line and $7^{th}$ line describe the array $C$. The $6^{th}$ line contains $N$ intergers $C_1$, $C_2$, $\cdots$, $C_N$, indicating the elements in array $C$. The $7^{th}$ line contains $N$ intergers $w_1$, $w_2$, $\cdots$, $w_N$, and $w_i$ is $1$ if $C_i$ is $changeable$,otherwise $w_i$ is $0$.<br><br>$\cdot$ $1 \leq T \leq 2000$.<br><br>$\cdot$ for 75% data, $1 \leq N \leq 10$.<br><br>$\cdot$ for 95% data, $1 \leq N \leq 50$.<br><br>$\cdot$ for 100% data, $1 \leq N \leq 100$.<br><br>$\cdot$ $0 \leq A_i, B_i, C_i \leq 10^9$.<br><br>$\cdot$ both $u_i, v_i, w_i$ is either $0$ or $1$.<br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>For every test case, you should output &quot;<b>Case #x: y</b>&quot;, where $x$ indicates the case number and counts from $1$, and $y$ is the winner of the game.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">3
2
4 3
1 1
4 4
0 1
5 5
0 0
2
4 4
1 1
4 4
0 0
5 5
0 0
2
4 4
1 1
4 4
0 0
4 4
0 0</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Output</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">Case #1: Tweek
Case #2: Craig
Case #3: Tweek</div></pre></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Source</div> <div class=panel_content> <a href="/search.php?field=problem&key=2015ACM%2FICPC%D1%C7%D6%DE%C7%F8%C9%CF%BA%A3%D5%BE-%D6%D8%CF%D6%C8%FC%A3%A8%B8%D0%D0%BB%BB%AA%B6%AB%C0%ED%B9%A4%A3%A9&source=1&searchmode=source"> 2015ACM/ICPC亚洲区上海站-重现赛（感谢华东理工） </a> </div> <div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Recommend</div> <div class=panel_content>wange2014</div><div class=panel_bottom>&nbsp;</div><br><center style='font-size:15px;font-family:Arial;font-weight:bold;color:#1A5CC8'><a href='statistic.php?pid=5579'>Statistic</a>&nbsp;|&nbsp;<a href='submit.php?pid=5579'>Submit</a>&nbsp;|&nbsp;<a href="./discuss/problem/list.php?problemid=5579">Discuss</a> | <a href='note/note.php?pid=5579'>Note</a><br></td></tr><tr>
