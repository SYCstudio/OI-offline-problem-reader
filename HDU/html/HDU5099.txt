<tr><td align=center><h1 style='color:#1A5CC8'>Comparison of Android versions</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 2358&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 975<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>   As an Android developer, itˇs really not easy to figure out a newer version of two kernels, because Android is updated so frequently and has many branches. Fortunately, Google identifies individual builds with a short build code, e.g. FRF85B.<br><br>   The first letter is the code name of the release family, e.g. F is Froyo. The code names are ordered alphabetically. The latest code name is K (KitKat).<br>The second letter is a branch code that allows Google to identify the exact code branch that the build was made from, and R is by convention the primary release branch.<br><br>   The next letter and two digits are a date code. The letter counts quarters, with A being Q1 2009. Therefore, F is Q2 2010. The two digits count days within the quarter, so F85 is June 24 2010.<br><br>   Finally, the last letter identifies individual versions related to the same date code, sequentially starting with A; A is actually implicit and usually omitted for brevity.<br><br>   Please develop a program to compare two Android build numbers.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>   The first line is an integer n (1 &lt;= n &lt;= 2000), which indicates how many test cases need to process.<br><br>   Each test case consists of a single line containing two build numbers, separated by a space character. </div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>   For each test case, output a single line starting with ¨Case #: 〃 (# means the number of the test case). Then, output the result of release comparison as follows:<br><br>   ●  Print &quot;&lt;&quot; if the release of the first build number is lower than the second one;<br>   ●  Print &quot;=&quot;  if the release of the first build number is same as he second one;<br>   ●  Print &quot;&gt;&quot;  if the release of the first build number is higher than the second one.<br><br>   Continue to output the result of date comparison as follows:<br>   ●  Print &quot;&lt;&quot;  if the date of the first build number is lower than the second one;<br>   ●  Print &quot;=&quot;  if the date of the first build number is same as he second one;<br>   ●  Print &quot;&gt;&quot;  if the date of the first build number is higher than the second one.<br><br>   If two builds are not in the same code branch, just compare the date code; if they are in the same code branch, compare the date code together with the individual version.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">2
