<tr><td align=center><h1 style='color:#1A5CC8'>Information Extraction</h1><font><b><span style='font-family:Arial;font-size:12px;font-weight:bold;color:green'>Time Limit: 2000/1000 MS (Java/Others)&nbsp;&nbsp;&nbsp;&nbsp;Memory Limit: 32768/32768 K (Java/Others)<br>Total Submission(s): 90&nbsp;&nbsp;&nbsp;&nbsp;Accepted Submission(s): 39<br></span></b></font><br><br><div class=panel_title align=left>Problem Description</div> <div class=panel_content>   Extracting the information from HTML document is a complex task. For example, journalists often need to extract some news from Web site (including the title, content, time, etc.), and then made it into a specific format. Because different HTML document has different structure and the artificial way to copy and paste is too tedious, now you need to write a program to store a specific output format, the structure of some HTML documents and the mapping from the structure to the output format. When it inputs a HTML document, according to the mapping, you need to output specific format text.<br>HTML documents are defined as follows：<br><b>HTML</b><br>　　HTML stands for HyperText Markup Language.<br>　　HTML is a markup language.<br>　　A markup language is a set of markup tags.<br>　　The tags describe document content.<br>　　HTML documents consist of tags and texts.<br><b>Tags</b><br>　　HTML is using tags for its syntax.<br>　　A tag is composed with special characters: ‘&lt;’, ‘&gt;’ and ‘/’.<br>　　Tags usually come in pairs, the opening tag and the closing tag.<br>　　The opening tag starts with “&lt;” and the tagname. It usually ends with a “&gt;”.<br>　　The closing tag starts with “&lt;/” and the same tagname as the corresponding opening tag. It ends with a “&gt;”.<br>　　There will not be any other angle brackets in the documents.<br>　　Tagnames are strings containing only lowercase letters.<br>　　Tags will contain no line break (‘\n’).<br>　　Except tags, anything occured in the document is considered as text content.<br>      The length of tagname less than or equal 30<br><b>Elements</b><br>　　An element is everything from an opening tag to the matching closing tag (including the two tags).<br>　　The element content is everything between the opening and the closing tag.<br>　　Some elements may have no content. They’re called empty elements, like &lt;hr&gt;&lt;/hr&gt;.<br>　　Empty elements can be closed in the opening tag, ending with a “/&gt;” instead of “&gt;”.<br>　　All elements are closed either with a closing tag or in the opening tag.<br>　　Elements can have attributes.<br>　　Elements can be nested (can contain other elements).<br>　　The &lt;html&gt; element is the container for all other elements, it will not have any attributes.<br><b>Attributes</b><br>　　Attributes provide additional information about an element.<br>　　Attributes are always specified in the opening tag after the tagname.<br>　　Tag name and attributes are separated by single space.<br>　　An element may have several attributes.<br>　　Attributes come in name=&quot;value&quot; pairs like class=&quot;icpc&quot;.<br>　　There will not be any space around the '='.<br>　　All attribute names are in lowercase.<br>      The value of the id attribute is unique and the length less than or equal 30.<br><b>A Simple Example</b><br>&lt;html&gt;&lt;body&gt;<br>    &lt;h3 id=&quot;header&quot; class=&quot;style1&quot;&gt;this is a test&lt;/h3&gt;<br>    &lt;div id=&quot;content&quot; class=&quot;style2&quot;&gt;<br>        this is content&lt;br/&gt;<br>        &lt;pre&gt;var x = 1111; &lt;/pre&gt;<br>    &lt;/div&gt;<br>&lt;/body&gt;&lt;/html&gt;<br>The structure of a HTML document is the HTML document, but only have id attribute and elements have no text content. A HTML document may have many structures, because the content of some elements may be removed.<br><b>A Simple Example</b><br>&lt;html&gt;&lt;body&gt;<br>    &lt;h3 id=&quot;header&quot;&gt;&lt;/h3&gt;<br>    &lt;div id=&quot;content&quot;&gt;&lt;/div&gt;<br>&lt;/body&gt;&lt;/html&gt;<br>The specific output format like a HTML document, but the container for all other elements is not necessarily the &lt;html&gt; element.<br><b>A Simple Example</b><br>&lt;news&gt;<br>    &lt;title&gt;&lt;/title&gt;<br>    &lt;content&gt;&lt;/content&gt;<br>    &lt;flag&gt; others &lt;/flag&gt;<br>&lt;/news&gt;<br>The mapping from the structure to the output format is defined as follows：<br>The value of a id attribute of a structure-A tagname of the ouput format <br>(It means the content of the element which the id belongs to should as the content of elements whose tagname is the tagname)<br><b>Two Simple Examples</b><br>   header-title<br>   content-content</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>   The first line of the input is an integer T (T&lt;=15) representing the number of test cases.<br>   Each test case is a specific output format (Length less than or equal 10000) in front.<br>   Then input is an integer N (0&lt;=N&lt;=30) representing the number of the type of HTML document.<br>   Each type is the structure of HTML document in front.<br>   Then input is an integer M (0&lt;=M&lt;=30) representing the number of the mapping from the structure to the specific output format. Each of the next M lines is a mapping.<br>   Each test case is a HTML document (Length less than or equal 10000) at the end.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>   For each test case, first output a line “Case #x:”, where x is the case number (starting from 1). If there exists the structure of the HTML doument, output specific format text, otherwise output “Can't Identify”. If there exists more than one structure of the HTML doument, use the early input structure.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">2
