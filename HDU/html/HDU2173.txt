To win, the player must produce the actual retail price of the car from a board of possible numbers like:<br><br><img style='max-width:100%;' src=/data/images/C127_1010_1.jpg><br><br> The player selects one number from each column to form a bid.  Using the above board as an example, the first number in the price of the car is either 1 or 3; the second is one of 9, 4, 2; the third is one of 0, 6, 8, 4; and so on.  Numbers may never move to a different column.<br><br>After the player selects their bid, Drew Carey lights up the numbers in the bid which are correct.  If the player has no numbers correct, the game ends and they lose; if they have at least one number correct, the game continues.<br><br>When the game continues, the player is given another opportunity to select numbers from those columns that were incorrect.  They will cover up the wrong bid numbers with different selections from the same columns.  Again, Drew Carey will light up any new correct digits.  If the player has no new numbers correct, the game ends and they lose; if they have at least one new number correct, the game continues.<br><br>For example:<br><br><br><img style='max-width:100%;' src=/data/images/C127_1010_2.jpg><br><br>The player selects an initial bid of $14677.  The 1, 4 and first 7 are correct (c stands for correct, x for incorrect, and v designates a column that requires a subsequent selection in the example above).  The player covers up the incorrect 6 and 7 with an 8 and a 3 for a second bid of $14873.  The 3 is correct, but the 8 is wrong.  At this point it's a 50/50 chance.  The player will select the 4 or the 0 and either win the car or lose the game.<br><br>The show's sponsors would like to know how frequently their cars are given away.  You are to use the assumption that players choose numbers uniformly from those remaining.<br><br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Input</div> <div class=panel_content>The sponsor will explore many variations, with prices up to 7 digits long.  Therefore, the input file will begin with a line containing the integer N &lt;= 5000, the number of test cases to be explored.  The test cases follow.<br><br>Each test case begins with the integer d, 0 &lt; d &lt;= 7, the number of digits in the price of the car.  d lines will follow, with non-empty strings of *distinct* digits in the range from 0 through 9.  Each of these lines represents a *column* of the digits in the game.  The first line represents the leftmost column; the last the rightmost column.  A 0 is possible as the first digit in the price of the car.<br><br></div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Output</div> <div class=panel_content>Your program will print the probability of the player winning the car, rounded to 3 decimals.</div><div class=panel_bottom>&nbsp;</div><br><div class=panel_title align=left>Sample Input</div><div class=panel_content><pre><div style="font-family:Courier New,Courier,monospace;">2
